# Конспект: МСФО 17, Резерв убытков, Рабочая документация аудитора

---

## ЧАСТЬ 1. МСФО (IFRS) 17 — Договоры страхования

### 1.1 Общее: что такое МСФО 17 и зачем он нужен

МСФО 17 (вступил в силу с 1 января 2023 г.) — стандарт, регулирующий **признание, оценку и раскрытие** договоров страхования.

**Заменил МСФО 4** (IFRS 4), который был временным стандартом и позволял компаниям применять локальные правила.

**Ключевая цель:** дать пользователям финансовой отчётности прозрачную и сопоставимую информацию о прибыльности страхового бизнеса.

**Кто применяет:** страховые компании, банки с подразделениями страхования, перестраховщики.

---

### 1.2 Сфера применения

МСФО 17 применяется к:
- Договорам страхования (в т.ч. перестрахования), выпущенным компанией
- Договорам перестрахования в активах (ceded reinsurance)
- Инвестиционным договорам с условиями дискреционного участия (DPF)

**Не применяется к:**
- Гарантиям по финансовым инструментам (→ МСФО 9)
- Гарантиям работодателя по пенсиям (→ МСФО 19)
- Гарантийным обязательствам производителей (→ МСФО 15)

---

### 1.3 Ключевые понятия

| Термин | Определение |
|---|---|
| **Группа договоров** | Портфель → когорта → группа (прибыльные / убыточные / нейтральные) |
| **Граница договора** | Период, в течение которого компания несёт обязательства и получает премии |
| **CSM (Contractual Service Margin)** | Незаработанная прибыль по договору; амортизируется по мере оказания услуг |
| **RA (Risk Adjustment)** | Компенсация за неопределённость нефинансовых рисков |
| **FCF (Fulfilment Cash Flows)** | Оценка будущих денежных потоков + RA |
| **BBA (Building Block Approach)** | Основной подход: FCF + CSM |
| **PAA (Premium Allocation Approach)** | Упрощённый подход (аналог unearned premium) для коротких договоров (≤1 года) |
| **VFA (Variable Fee Approach)** | Для договоров с прямым участием; CSM отражает долю в активах |

---

### 1.4 Три модели оценки

#### Модель 1: BBA (General Model / Общий подход)

```
Обязательство по страховому покрытию =
    Оценка будущих CF (ожидаемые) + RA (надбавка за риск) + CSM
```

**Этапы:**
1. **Признание** — при начале договора CSM = −(FCF), т.е. = PV[будущие поступления] − PV[будущие выплаты и расходы] − RA
2. **Последующая оценка** — каждый период:
   - CSM корректируется на изменения в нефинансовых допущениях
   - Финансовые изменения (ставка дисконтирования) → в OCI или P&L (учётная политика)
   - CSM амортизируется → страховая выручка (Insurance Revenue)
3. **Убыточные договоры** — CSM = 0; убыток признаётся немедленно как Loss Component

#### Модель 2: PAA (Упрощённый подход)

- Применяется, если срок покрытия ≤ 12 месяцев **или** результат не отличается существенно от BBA
- Обязательство = незаработанная премия (аналогично МСФО 4)
- При наступлении страхового случая → оценка по BBA (или упрощённо)

#### Модель 3: VFA (Variable Fee Approach)

- Для договоров unit-linked / участие в прибыли
- CSM = доля страхователя в базовых активах − FCF
- Изменения стоимости активов → корректируют CSM, а не P&L

---

### 1.5 Компоненты страховой выручки (Insurance Revenue)

По МСФО 17 **премии ≠ выручка**. Выручка признаётся по мере оказания страхового покрытия:

```
Insurance Revenue =
  Ожидаемые страховые выплаты и расходы (за период)
+ Амортизация RA
+ Амортизация CSM
+ Прочие расходы на обслуживание
```

**Insurance Finance Income/Expense** — отдельная строка: эффект временной стоимости денег и финансовых рисков.

---

### 1.6 Раскрытия по МСФО 17

Обязательные таблицы:
- **Reconciliation CSM** — движение CSM за период
- **Reconciliation Liabilities** — движение обязательств (LRC + LIC)
- **Claims development tables** — развитие убытков по когортам
- **Sensitivity analysis** — чувствительность к ключевым допущениям

---

### 1.7 Переход на МСФО 17

Три метода перехода:
| Метод | Суть | Когда применяется |
|---|---|---|
| **Полный ретроспективный** | Пересчёт с даты выпуска договора | Если данные доступны |
| **Модифицированный ретроспективный** | Упрощения при расчёте CSM | Если полный ретроспективный неосуществим |
| **Метод справедливой стоимости** | CSM = FV − FCF на дату перехода | Если данных нет совсем |

## ЧАСТЬ 2. Оценка резерва убытков (Loss Reserve / IBNR)

### 2.1 Виды страховых резервов

| Резерв | Что покрывает | Аббревиатура |
|---|---|---|
| **Резерв незаработанной премии** | Часть премии, относящаяся к будущему периоду | UPR / НРП |
| **Резерв заявленных, но неурегулированных убытков** | Известные случаи, ещё не выплаченные | RBNS / РЗНУ |
| **Резерв произошедших, но незаявленных убытков** | Случаи уже произошли, но не заявлены | IBNR / РПНУ |
| **Стабилизационный резерв** | Сглаживание крупных/катастрофических выплат | — |

---

### 2.2 IBNR (Incurred But Not Reported)

**IBNR = оценка будущих выплат** по убыткам, которые:
- уже произошли на отчётную дату,
- но ещё не заявлены страхователем **или**
- заявлены, но оценка неполная (IBNER — Incurred But Not Enough Reserved).

**Почему важен:** занижение IBNR = завышение прибыли и капитала → регуляторные/аудиторские риски.

---

### 2.3 Методы оценки IBNR

#### Метод 1: Chain-Ladder (Развитие убытков / Development Method)

**Идея:** будущее развитие убытков повторяет историческое.

**Шаги:**
1. Строим **треугольник развития убытков** (Loss Development Triangle):
   - Строки = годы возникновения (accident year)
   - Столбцы = периоды развития (12, 24, 36... месяцев)
2. Вычисляем **факторы развития** (LDF — Loss Development Factors):
   ```
   LDF(n→n+1) = Σ убытки на шаге n+1 / Σ убытки на шаге n
   ```
3. Проецируем незавершённые когорты до **хвостового коэффициента** (tail factor → 1.0)
4. **IBNR = Projected Ultimate − Current Paid/Incurred**

**Пример треугольника:**

| AY \ Dev | 12 мес | 24 мес | 36 мес | Ultimate |
|---|---|---|---|---|
| 2021 | 1 000 | 1 400 | 1 500 | 1 500 |
| 2022 | 1 200 | 1 650 | → | 1 785 |
| 2023 | 1 100 | → | → | 1 320 |

LDF(12→24) = (1400+1650)/(1000+1200) = **1,386**
LDF(24→36) = 1500/1400 = **1,071**

#### Метод 2: Bornhuetter-Ferguson (BF)

**Идея:** комбинирует актуарную оценку «a priori» с фактическими данными.

```
Ultimate = Paid + (Expected Ultimate × % неразвитого убытка)
```

- Используется, когда данных мало (молодые когорты)
- Менее чувствителен к выбросам, чем Chain-Ladder

#### Метод 3: Expected Loss Ratio (ELR)

```
IBNR = Заработанная премия × ELR − Paid Losses
```

- Применяется при отсутствии достаточной истории
- ELR берётся из андеррайтерской оценки или отраслевых данных

#### Метод 4: Cape Cod

- Комбинирует ELR из фактических данных с Chain-Ladder
- ELR = Paid Losses / (Earned Premium × % развитого убытка)

---

### 2.4 Ключевые допущения и чувствительность

| Допущение | Влияние на резерв |
|---|---|
| Выбор периода для LDF | Длинный период → стабильнее, но медленно реагирует на тренды |
| Tail factor | Критичен для долгосрочных линий (liability, workers comp) |
| Mix of methods | Взвешивание по надёжности данных |
| Инфляция убытков (claims inflation) | Без корректировки → занижение резерва |

---

### 2.5 Резерв убытков в банке (кредитные риски)

В банковском контексте аналог — **резерв под обесценение кредитов (ECL — Expected Credit Loss)** по МСФО 9:

| Стадия | Признак | Резерв |
|---|---|---|
| **Stage 1** | Нет значительного роста кредитного риска | 12-месячный ECL |
| **Stage 2** | Значительный рост риска, но не дефолт | Lifetime ECL |
| **Stage 3** | Дефолт / обесценение | Lifetime ECL (индивидуальная оценка) |

**Компоненты ECL:**
```
ECL = PD × LGD × EAD × Дисконтирующий фактор
```
- **PD** — вероятность дефолта
- **LGD** — уровень потерь в случае дефолта (1 − Recovery Rate)
- **EAD** — величина под риском на момент дефолта

**Аудиторские риски по ECL:**
- Корректность сегментации на стадии
- Обоснованность макроэкономических сценариев и весов
- Перенос из Stage 1 в Stage 2 (триггеры)
- Индивидуальная оценка крупных заёмщиков (Stage 3)

## ЧАСТЬ 3. Рабочая документация аудитора

### 3.1 Правовая база

- **МСА 230** (ISA 230) — «Аудиторская документация»: требования к составу, форме и хранению
- **МСА 315** — Понимание компании и оценка рисков
- **МСА 330** — Ответные действия на оцененные риски
- **МСА 500–580** — Аудиторские доказательства

---

### 3.2 Цели рабочей документации

1. **Доказать** выполнение аудиторских процедур
2. **Обосновать** выводы и аудиторское мнение
3. **Обеспечить контроль качества** (проверка руководителем группы, партнёром, QCRE)
4. **Обеспечить преемственность** при смене команды
5. **Выполнить регуляторные требования** (хранение ≥5 лет; для публичных компаний — 7+ лет)

---

### 3.3 Структура аудиторского файла

```
Аудиторский файл
├── Постоянный файл (Permanent File)
│   ├── Учредительные документы, лицензии
│   ├── Договор на оказание аудиторских услуг
│   ├── Оценка независимости
│   └── Ключевые учётные политики (стабильные)
│
└── Текущий файл (Current File)
    ├── A. Планирование
    │   ├── A1. Стратегия аудита (Overall Audit Strategy)
    │   ├── A2. Уровень существенности (Materiality)
    │   ├── A3. Оценка рисков (Risk Assessment — RCM)
    │   └── A4. План аудита (Audit Plan)
    ├── B. Понимание бизнеса и среды контроля
    │   ├── B1. Бизнес-профиль клиента
    │   ├── B2. ИТ-среда и общие ИТ-контроли (GITC)
    │   └── B3. Операционные риски
    ├── C–Z. Разделы по статьям ФО (по сегментам/счетам)
    │   ├── Cx. Кредитный портфель / ECL
    │   ├── Dx. Страховые обязательства / МСФО 17
    │   └── ...
    └── ZZ. Завершение аудита
        ├── ZZ1. Последующие события (МСА 560)
        ├── ZZ2. Непрерывность деятельности (МСА 570)
        ├── ZZ3. Письмо-представление руководства (МСА 580)
        └── ZZ4. Аудиторское заключение
```

---

### 3.4 Уровень существенности

| Показатель | Типичные бенчмарки |
|---|---|
| Общая (Planning) Materiality | 5% PBT / 0,5–1% выручки / 1% активов / 1% капитала |
| Performance Materiality | 50–75% от общей существенности |
| Trivial / Clearly Inconsequential | 3–5% от общей существенности |

**Для банков чаще используют:**
- % от суммарных активов или чистых активов
- % от чистой процентной маржи (NII)

**Для страховщиков:**
- % от заработанной премии или технических резервов

---

### 3.5 Оценка рисков (Risk & Control Matrix — RCM)

Структура рабочего документа по рискам:

| Поле | Содержание |
|---|---|
| **Бизнес-процесс** | Напр.: «Урегулирование убытков» |
| **Риск** | Что может пойти не так (утверждения ACERVO) |
| **Утверждение** | Completeness / Accuracy / Existence / Rights / Valuation / Occurrence / Cut-off |
| **Контроль клиента** | Описание контрольной процедуры |
| **Тип контроля** | Preventive / Detective; Manual / Automated |
| **Наш ответ** | Test of Controls (ToC) или Substantive Procedures (SP) |
| **Ссылка на РД** | Номер рабочего документа |
| **Вывод** | Контроль эффективен / не эффективен |

---

### 3.6 Рабочий документ по существу (Substantive Workpaper)

**Обязательные элементы каждого РД:**

```
Заголовок: [Название статьи / процедуры]
Период: [Отчётный период]
Подготовил: [Имя, дата]
Проверил: [Имя руководителя, дата]
Цель: [Что проверяем и почему]
Утверждения: [напр.: Valuation, Completeness]
─────────────────────────────────────────────
[Описание процедуры и источника данных]
[Результаты тестирования / таблица]
[Выявленные отклонения (exceptions)]
─────────────────────────────────────────────
Вывод: [Достаточно ли доказательств; есть ли замечания]
Ссылки: [TB ref, управленческие письма, прочие РД]
```

### 3.7 Типовые процедуры для резервов (банк/страховщик)

#### Аудит ECL / резерва под обесценение кредитов (банк)

| Процедура | Что проверяем |
|---|---|
| Проверка модели ECL (Model Audit) | Корректность формул PD, LGD, EAD; независимое воспроизведение расчёта |
| Staging test | Правильность отнесения к Stage 1/2/3 по 30/90 дней просрочки и качественным триггерам |
| Macro overlay | Обоснованность макросценариев (базовый / оптимистичный / пессимистичный) и их весов |
| Индивидуальная оценка (Stage 3) | Оценка залогов, перспектив реструктуризации, cash flow от заёмщика |
| Data integrity | Согласование данных из кредитной системы с расчётом ECL |
| Back-testing | Сравнение прогнозных и фактических потерь предыдущих когорт |

#### Аудит страховых резервов (МСФО 17 / IBNR)

| Процедура | Что проверяем |
|---|---|
| Привлечение актуарного специалиста | Оценка методологии, допущений, полноты данных |
| Независимое воспроизведение Chain-Ladder | Пересчёт LDF и IBNR по ключевым линиям |
| Анализ чувствительности | Как изменяется резерв при ±10% LDF, ±1% ELR |
| Сверка с Triangle Data | Данные в треугольнике = данные в системах клиента |
| Проверка хвостового коэффициента | Обоснованность tail factor для «длинных хвостов» |
| Adequacy test | Резерв ≥ BEL (Best Estimate Liability) по требованию регулятора |
| Сравнение с предыдущим годом | Изменение методологии или допущений — обоснование |

---

### 3.8 Типовые замечания (findings) и как их документировать

| Тип | Пример | Оформление в РД |
|---|---|---|
| **Misstatement** | Резерв занижен на X млн руб. | Форма Summary of Misstatements (SAM) |
| **Disclosure deficiency** | Нет раскрытия по sensitivity МСФО 17 | Письмо менеджменту (Management Letter) |
| **Control deficiency** | Отсутствует независимая проверка IBNR-расчёта | Reportable Deficiency / Material Weakness |
| **Going concern indicator** | Коэффициент покрытия резервов < 100% | Документация в разделе ZZ2 |

---

### 3.9 Ключевые требования МСА 230

- **Своевременность**: РД составляются в ходе аудита, не постфактум
- **Полнота**: РД должны позволить опытному аудитору воспроизвести выводы без дополнительных объяснений
- **Изменения после даты заключения**: допускаются только с обоснованием и подписью партнёра
- **Конфиденциальность**: файл не передаётся клиенту без явного согласования

---

### 3.10 Особенности аудита банка

| Область | Специфика |
|---|---|
| **Регуляторные требования** | Нормативы ЦБ (Н1, Н2, Н3, Н4), РСБУ + МСФО одновременно |
| **Объём операций** | Тысячи кредитных договоров → выборочный подход + аналитика |
| **ИТ-зависимость** | Расчёт ECL почти всегда автоматизирован → обязательный GITC-аудит |
| **Связанные стороны** | Кредиты акционерам и аффилированным структурам — повышенный риск |
| **Ликвидность** | Анализ разрывов ликвидности (GAP-анализ) — риск непрерывности |
| **Торговый портфель** | Оценка FV по МСФО 9; уровни иерархии FV (L1/L2/L3) |
