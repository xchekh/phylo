# ALM — Asset-Liability Management (Управление активами и пассивами)

## Что такое ALM

ALM — процесс управления финансовыми рисками, возникающими из-за несоответствия активов и пассивов по срокам, валютам, процентным ставкам и денежным потокам.

Цель: обеспечить устойчивость банка/страховщика/пенсионного фонда при изменении рыночной конъюнктуры, при этом не жертвуя доходностью.

**Ключевой принцип:** активы и пассивы рассматриваются не по отдельности, а как единая система. Риск возникает не из-за одной стороны баланса, а из-за их взаимодействия.

---

## Основные риски в ALM

| Риск | Описание |
|------|----------|
| **Процентный риск (IRR)** | Изменение ставок влияет по-разному на активы и пассивы → NII и EVE меняются |
| **Риск ликвидности** | Не хватает денег вовремя расплатиться по обязательствам |
| **Валютный риск** | Активы и пассивы в разных валютах — курс двигается против позиции |
| **Кредитный риск** | Дефолт по активам при сохранении обязательств |
| **Риск досрочного погашения (prepayment)** | Заёмщики гасят кредиты досрочно при падении ставок |
| **Риск модели** | Неточность допущений о поведении клиентов и денежных потоков |

---

## Ключевые метрики ALM

### 1. NII — Net Interest Income (Чистый процентный доход)
```
NII = Процентные доходы по активам − Процентные расходы по пассивам
```
- Краткосрочная метрика (обычно горизонт 1–2 года)
- Показывает, сколько зарабатывает банк на спреде прямо сейчас
- Чувствителен к изменению ставок на ближних инструментах

### 2. EVE — Economic Value of Equity (Экономическая стоимость капитала)
```
EVE = PV(Активы) − PV(Пассивы)
```
- Долгосрочная метрика — оценивает влияние ставок на всю жизнь инструментов
- Показывает, что останется акционерам при немедленной ликвидации по рыночным ценам
- Более волатилен, чем NII при шоках дальней части кривой

### 3. Gap-анализ (Разрывы)
```
Gap(t) = Активы, переоцениваемые в период t − Пассивы, переоцениваемые в период t
```
- Положительный Gap → банк выигрывает при росте ставок (активы переоцениваются быстрее)
- Отрицательный Gap → выигрывает при падении ставок

**Виды Gap:**
- **Repricing Gap** — разрыв по дате переоценки ставки
- **Maturity Gap** — разрыв по сроку погашения
- **Duration Gap** — разрыв по дюрациям (взвешенный)

### 4. Duration и Modified Duration
```
Duration (Macaulay) = Σ [t × CF(t) / (1+r)^t] / P

Modified Duration = Macaulay Duration / (1 + r)

ΔP / P ≈ −ModD × Δr
```
- Мера чувствительности цены к изменению ставки
- Duration Gap = Duration(Активы) − Duration(Пассивы) × (Пассивы / Активы)

### 5. DV01 / BPV (Basis Point Value)
```
DV01 = −ΔP при сдвиге ставки на 1 б.п. (0.01%)
```
- Показывает рублёвую / долларовую чувствительность портфеля к 1 б.п.
- Используется для хеджирования: нужно выйти на нулевой DV01 по портфелю

### 6. Коэффициент LCR (Liquidity Coverage Ratio)
```
LCR = HQLA / Чистый отток денег за 30 дней ≥ 100%
```
- HQLA — высоколиквидные активы (кэш, госбумаги)
- Регуляторный минимум Basel III

### 7. NSFR (Net Stable Funding Ratio)
```
NSFR = Доступное стабильное финансирование / Требуемое стабильное финансирование ≥ 100%
```
- Долгосрочная метрика ликвидности (горизонт 1 год)

## Инструменты ALM

### I. Gap-анализ и таблицы переоценки

**Что это:** Матрица, где строки — временны́е бакеты (до 1 мес, 1–3 мес, 3–6 мес, ..., 5+ лет), столбцы — активы / пассивы / Gap.

**Для чего:**
- Найти временны́е разрывы между активами и пассивами
- Понять, когда банк уязвим к росту/падению ставок
- Исходная точка для хеджирования

**Ограничения:** не учитывает нелинейности, поведение клиентов (депозиты до востребования, досрочное погашение)

---

### II. Стресс-тестирование процентного риска

**Стандартные шоки (Basel / ЦБ РФ):**
- Параллельный сдвиг кривой ±200 б.п.
- Крутизна: long up / short down
- Short rate shock (шок коротких ставок)
- Инверсия кривой

**Что считают:** изменение NII и EVE при каждом шоке → если потери > порог капитала, нужно хеджировать.

**Behavioural assumptions (поведенческие допущения):**
- Средний срок жизни депозита до востребования (Core Deposits)
- Коэффициент досрочного погашения ипотеки (CPR — Conditional Prepayment Rate)

---

### III. Процентные свопы (Interest Rate Swaps, IRS)

**Структура:**
- Банк платит фиксированную ставку → получает плавающую (Pay Fixed Swap)
- Банк платит плавающую → получает фиксированную (Receive Fixed Swap)

**Применение в ALM:**
- У банка длинные фиксированные активы (ипотека) и короткие пассивы → Duration Gap положительный
- Для хеджирования: Pay Fixed Swap (сокращает дюрацию активов / увеличивает дюрацию пассивов)
- Нет передачи права собственности, только обмен денежными потоками

**Пример:**
```
Банк держит ипотеку под 10% фиксированных на 20 лет.
Фондируется депозитами под RUONIA + 1% (плавающие, короткие).
При росте RUONIA расходы растут, доходы не меняются → NII падает.
Решение: Pay Fixed Swap, где банк платит 10% и получает RUONIA → синтетически превращает ипотеку во флоатер.
```

---

### IV. Кросс-валютные свопы (Cross-Currency Swaps, XCCY)

**Структура:** Обмен номиналов + денежных потоков в двух разных валютах.

**Применение:**
- Банк привлёк евровые депозиты, но хочет финансировать рублёвые активы
- XCCY своп: получает EUR (выплачивает вкладчикам) → отдаёт RUB (финансирует активы)
- Хеджирует валютный Gap

---

### V. Опционы на процентные ставки

**Caps и Floors:**
- **Cap** = серия колл-опционов на ставку. Покупатель защищён от роста ставки выше страйка. Банк покупает cap, если у него плавающие пассивы и страх роста ставок.
- **Floor** = серия пут-опционов. Защита от падения ставки ниже уровня. Банк покупает floor, если у него плавающие активы.
- **Collar** = Cap + Floor (купленный + проданный) → ограничивает диапазон ставки за меньшую премию

**Swaptions:**
- Опцион на вход в процентный своп
- Payer swaption: право заплатить фиксированную → используют, если ожидают рост ставок, но не уверены
- Применяется для хеджирования досрочного погашения ипотеки (закрыли своп → нужна возможность его расторгнуть)

---

### VI. Репо и обратное репо (Repo / Reverse Repo)

**Что это:** Продажа актива с обязательством обратного выкупа (фактически — обеспеченный кредит).

**Применение в ALM:**
- Управление краткосрочной ликвидностью
- Пополнение HQLA буфера
- Привлечение дешёвого фондирования под залог госбумаг

---

### VII. FRA — Forward Rate Agreement (Соглашение о будущей ставке)

**Что это:** OTC-контракт, фиксирующий ставку на будущий период.

**Пример:** FRA 3×6 — ставка на период с 3-го по 6-й месяц фиксируется сегодня.

**Применение:** Хеджирование риска переоценки краткосрочных позиций. Более простой инструмент, чем своп, для коротких горизонтов.

---

### VIII. Фьючерсы на процентные ставки

**Примеры:** Futures на RUONIA, Eurodollar futures (SOFR futures), Bund futures.

**Применение:**
- Хеджирование коротких позиций по ставкам (ликвидно, стандартизировано)
- Построение форвардной кривой для оценки EVE / NII

---

### IX. Секьюритизация (Securitization)

**Что это:** Продажа пула активов (ипотека, автокредиты) в SPV, который выпускает ценные бумаги (ABS, MBS, CLO).

**Применение в ALM:**
- Снижение Duration Gap (избавляемся от длинных активов)
- Высвобождение ликвидности
- Снижение нагрузки на капитал (если off-balance sheet)
- Передача prepayment-риска инвесторам

---

### X. Управление депозитной базой (Core Deposit Modelling)

**Проблема:** Депозиты до востребования формально — overnight, но фактически остаются годами. Если моделировать их как overnight, Duration Gap будет завышен.

**Инструменты:**
- Статистические модели поведения остатков (econometric / behavioural)
- Разделение на Core (стабильная часть, модельный срок 3–5 лет) и Non-Core (волатильная, срок = overnight)
- Чувствительность к ставке (beta): насколько банк передаёт рост/снижение ключевой ставки в ставку по депозиту

## ALM в банке: организационная структура

```
ALCO (Asset-Liability Committee)
  ├── Казначейство (Treasury) — исполнение, хеджирование, ликвидность
  ├── ALM-аналитика — Gap, EVE, NII, стресс-тесты
  ├── Риск-менеджмент — лимиты, мониторинг, валидация моделей
  └── Финансы / CFO — P&L, МСФО-эффект хеджей (IAS 39 / IFRS 9)
```

**ALCO** — ключевой орган принятия решений:
- Устанавливает аппетит к процентному риску
- Утверждает стратегию хеджирования
- Контролирует ликвидностную позицию
- Рассматривает стресс-сценарии

---

## Учёт хеджирования (Hedge Accounting — IFRS 9)

Без hedge accounting: производный инструмент переоценивается через P&L → волатильность прибыли.

С hedge accounting: переоценка хеджа и хеджируемого инструмента отражаются симметрично → P&L сглаживается.

**Виды:**
- **Fair Value Hedge** — хедж изменения справедливой стоимости (напр., фиксированная ставка по активу)
- **Cash Flow Hedge** — хедж изменения денежных потоков (напр., плавающая ставка по пассиву)
- **Net Investment Hedge** — хедж валютного риска зарубежного подразделения

**Условия применения:**
1. Формальная документация хеджа
2. Экономическое соответствие (economic relationship)
3. Эффективность хеджа (prospective / retrospective test)

## Практическая логика ALM-решения

**Шаг 1:** Построить карту денежных потоков — когда что приходит и уходит.

**Шаг 2:** Выявить Gap'ы — где активы и пассивы не совпадают по срокам / ставкам / валютам.

**Шаг 3:** Рассчитать чувствительность NII и EVE к шокам ставок.

**Шаг 4:** Сравнить с лимитами ALCO → нужно ли хеджировать?

**Шаг 5:** Выбрать инструмент хеджирования:
- Краткосрочный Gap → FRA, фьючерсы
- Долгосрочный Gap → IRS, XCCY
- Нелинейный риск (prepayment, caps) → опционы, swaptions
- Ликвидность → репо, секьюритизация

**Шаг 6:** Оформить hedge accounting (если нужна P&L-стабильность).

**Шаг 7:** Мониторинг и переоценка позиции на регулярной основе.

## Быстрые формулы для запоминания

```
NII sensitivity = Gap × Δr × (дней / 360)
EVE sensitivity ≈ −Duration Gap × Assets × Δr
DV01 портфеля = Σ DV01(инструмент_i)
LCR = HQLA / Net Cash Outflow (30 дн.) ≥ 100%
NSFR = ASF / RSF ≥ 100%
Cap payoff = max(r_market − Strike, 0) × Nominal × (дней / 360)
```
