# Финансовый анализ — полный конспект

## Что такое финансовый анализ

Финансовый анализ — процесс оценки финансового состояния, результатов деятельности и перспектив компании на основе её финансовой отчётности и других данных.

**Цели:**
- Оценить платёжеспособность и ликвидность
- Измерить доходность и эффективность бизнеса
- Выявить риски и финансовую устойчивость
- Принять решение об инвестиции, кредите, сделке M&A

**Кто использует:**
| Пользователь | Цель |
|---|---|
| Инвестор | Стоит ли покупать акции / облигации |
| Банк-кредитор | Может ли заёмщик обслуживать долг |
| M&A-аналитик | Сколько стоит компания для сделки |
| Менеджмент | Где теряем / зарабатываем |
| Регулятор | Финансовая устойчивость системообразующих компаний |
| Аудитор | Достоверность отчётности |

---

## Источники данных

| Документ | Что содержит |
|----------|-------------|
| **Баланс (Balance Sheet)** | Активы, обязательства, капитал — на дату |
| **ОПУ / P&L** | Доходы, расходы, прибыль — за период |
| **Отчёт о движении денег (CF Statement)** | Реальные денежные потоки — операционный, инвестиционный, финансовый |
| **Отчёт об изменении капитала** | Дивиденды, допэмиссии, buyback |
| **Примечания к отчётности** | Учётная политика, детализация статей, сегменты |
| **MD&A** | Комментарии менеджмента к результатам |
| **Отраслевые данные** | Для сравнения с конкурентами (benchmarking) |

---

## Виды финансового анализа

### По объекту
- **Анализ компании** — оценка конкретного эмитента / заёмщика
- **Отраслевой анализ** — сравнение компании с сектором
- **Макроанализ** — влияние экономики на бизнес

### По горизонту
- **Ретроспективный** — что было (факт)
- **Текущий** — что происходит сейчас
- **Прогнозный** — что будет (financial modeling, DCF)

### По методу
- **Горизонтальный** — динамика показателей во времени (год к году, квартал к кварталу)
- **Вертикальный** — структура: каждая строка как % от выручки (common size)
- **Коэффициентный (ratio analysis)** — расчёт и интерпретация финансовых коэффициентов
- **Сравнительный (benchmarking)** — сравнение с конкурентами и отраслью
- **Факторный** — разложение показателя на факторы (DuPont и др.)

## Блок 1. Анализ прибыльности (Profitability)

### Маржинальность

```
Валовая маржа (Gross Margin) = Валовая прибыль / Выручка
Валовая прибыль = Выручка − Себестоимость

EBITDA Margin = EBITDA / Выручка

EBIT Margin (Operating Margin) = EBIT / Выручка

Чистая маржа (Net Margin) = Чистая прибыль / Выручка
```

**Что показывает:** на каждый рубль выручки сколько остаётся после соответствующих затрат.

**Норма:** зависит от отрасли. Ритейл — 2–5% чистая маржа, IT — 20–30%, нефтянка — 10–15%.

### Рентабельность капитала

```
ROE = Чистая прибыль / Средний собственный капитал

ROA = Чистая прибыль / Средние активы

ROIC = NOPAT / Invested Capital
     = EBIT × (1 − t) / (Собственный капитал + Чистый долг)

ROCE = EBIT / Capital Employed
     = EBIT / (Активы − Краткосрочные обязательства)
```

**Ключевой принцип:** ROE > стоимость капитала (Ke) → компания создаёт стоимость. Если ROE < Ke → разрушает.

## Блок 2. Декомпозиция DuPont

### Трёхфакторная модель
```
ROE = Чистая маржа × Оборачиваемость активов × Финансовый рычаг

ROE = (Чистая прибыль / Выручка) × (Выручка / Активы) × (Активы / Капитал)
```

**Интерпретация:**
- Низкий ROE из-за маржи → проблема с ценообразованием / издержками
- Низкий ROE из-за оборачиваемости → активы работают неэффективно
- Низкий ROE из-за рычага → компания недолевая (не обязательно плохо)

### Пятифакторная модель
```
ROE = (Чистая прибыль / EBT) × (EBT / EBIT) × (EBIT / Выручка) × (Выручка / Активы) × (Активы / Капитал)
       Налоговое бремя        Процентное бремя   Операц. маржа   Оборачиваемость   Рычаг
```

## Блок 3. Анализ ликвидности

Ликвидность — способность компании погасить краткосрочные обязательства.

```
Коэффициент текущей ликвидности (Current Ratio) = Оборотные активы / Краткосрочные обязательства
  Норма: ≥ 2 (для производства), ≥ 1,5 (для торговли)

Быстрая ликвидность (Quick Ratio) = (Оборотные активы − Запасы) / Краткосрочные обязательства
  Норма: ≥ 1

Абсолютная ликвидность (Cash Ratio) = (Денежные средства + Краткосрочные фин. вложения) / Краткосрочные обязательства
  Норма: ≥ 0,2

Чистый оборотный капитал (NWC) = Оборотные активы − Краткосрочные обязательства
```

**Важно:** высокая ликвидность ≠ хорошо автоматически. Избыток кэша — потеря доходности. Дефицит — риск дефолта.

## Блок 4. Анализ финансовой устойчивости (Leverage / Solvency)

```
Долг / Капитал (D/E) = Общий долг / Собственный капитал
  Норма: зависит от отрасли. Для промышленности < 1, для банков — специфика

Долг / EBITDA = Чистый долг / EBITDA
  Норма: < 3x (комфортно), 3–5x (умеренно), > 5x (высокая нагрузка)
  Чистый долг = Долгосрочный долг + Краткосрочный долг − Денежные средства

Коэффициент покрытия процентов (ICR, Interest Coverage Ratio) = EBIT / Процентные расходы
  Норма: ≥ 3

Коэффициент автономии = Собственный капитал / Активы
  Норма: ≥ 0,5 (50% активов финансируется из собственных средств)

Коэффициент финансовой зависимости = Активы / Собственный капитал
  (обратный к автономии)
```

## Блок 5. Анализ деловой активности (Оборачиваемость)

```
Оборачиваемость активов = Выручка / Средние активы

Оборачиваемость запасов = Себестоимость / Средние запасы
Дни запасов (DIO) = 365 / Оборачиваемость запасов

Оборачиваемость дебиторки (DSO) = Дебиторская задолженность / (Выручка / 365)
  Сколько дней в среднем клиент платит

Оборачиваемость кредиторки (DPO) = Кредиторская задолженность / (Себестоимость / 365)
  Сколько дней компания сама тянет с оплатой поставщикам
```

### Цикл конвертации денег (CCC — Cash Conversion Cycle)
```
CCC = DIO + DSO − DPO
```
- Чем меньше CCC → тем меньше денег заморожено в оборотном капитале
- Отрицательный CCC (Amazon, Wildberries) — компания получает деньги от покупателей раньше, чем платит поставщикам

## Блок 6. Анализ денежных потоков

### Три раздела отчёта о ДДС
```
OCF (Operating Cash Flow) — денежный поток от операций
  = Чистая прибыль + Амортизация ± Изменение NWC

ICF (Investing Cash Flow) — инвестиционная деятельность
  = Capex + Приобретения M&A + Поступления от продажи активов

FCF (Free Cash Flow) — свободный денежный поток
  = OCF − Capex
  Показывает, сколько денег осталось после поддержания и развития бизнеса

FCFF (Free Cash Flow to Firm)
  = EBIT × (1-t) + D&A − Capex − ΔNWC
  Поток, доступный всем поставщикам капитала (кредиторам + акционерам)

FCFE (Free Cash Flow to Equity)
  = FCFF − Процентные расходы × (1-t) + Изменение долга
  Поток, доступный только акционерам
```

### Почему прибыль ≠ денежный поток
- Начисление выручки без поступления денег (дебиторка)
- Амортизация — расход в P&L, не движение денег
- Capex — движение денег, не расход в P&L сразу
- Изменение запасов / NWC

## Блок 7. Оценка стоимости компании (Valuation)

### 7.1 Метод DCF (Discounted Cash Flow)
```
Стоимость компании (EV) = Σ FCFF_t / (1 + WACC)^t + Terminal Value / (1 + WACC)^n

Terminal Value = FCFF_n × (1 + g) / (WACC − g)  [метод Gordon]
              или = EBITDA_n × EV/EBITDA_exit_multiple  [метод мультипликатора]

Стоимость акционерного капитала (Equity Value) = EV − Чистый долг
Цена акции = Equity Value / Количество акций
```

**WACC (Weighted Average Cost of Capital):**
```
WACC = Ke × E/(D+E) + Kd × (1-t) × D/(D+E)

Ke (стоимость акционерного капитала) = Rf + β × (Rm − Rf)  [CAPM]
Kd (стоимость долга до налогов) = ставка по кредитам/облигациям
```

**Плюсы DCF:** фундаментален, учитывает специфику компании.  
**Минусы:** очень чувствителен к допущениям (WACC, g); «мусор на входе — мусор на выходе».

### 7.2 Сравнительный анализ (Comps / Trading Multiples)

Оцениваем компанию по аналогии с рыночными мультипликаторами похожих компаний.

| Мультипликатор | Формула | Смысл |
|---|---|---|
| **EV/EBITDA** | Enterprise Value / EBITDA | Главный мультипликатор. Не зависит от структуры капитала и налогов |
| **EV/EBIT** | EV / EBIT | Учитывает амортизацию — важно для капиталоёмких отраслей |
| **EV/Revenue** | EV / Выручка | Для убыточных компаний или быстрорастущих |
| **P/E** | Цена акции / EPS | Рыночная оценка прибыли. Зависит от структуры капитала |
| **P/B** | Цена акции / Балансовая стоимость | Для банков и финансовых компаний |
| **P/S** | Рыночная капитализация / Выручка | Для стартапов и убыточных компаний |
| **EV/FCF** | EV / Free Cash Flow | Реальная денежная доходность |

**Enterprise Value:**
```
EV = Рыночная капитализация + Чистый долг + Доля меньшинства − Денежные средства
```

### 7.3 Метод сделок (Transaction Multiples / Precedent Transactions)
Аналогично comps, но берутся мультипликаторы из реальных M&A-сделок.
Обычно выше торговых мультипликаторов на **премию за контроль** (20–30%).

### 7.4 Метод чистых активов (NAV / Book Value)
```
NAV = Активы по рыночной стоимости − Обязательства
```
Применяется для: банков, страховщиков, холдингов, компаний в ликвидации.

### 7.5 Метод дивидендов (DDM — Dividend Discount Model)
```
P = D1 / (Ke − g)
```
Применяется для зрелых компаний со стабильными дивидендами.

## Блок 8. Кредитный анализ

Анализ заёмщика с точки зрения банка или держателя облигаций.

### Ключевые метрики
```
Долг / EBITDA — долговая нагрузка (leverage)
EBITDA / Процентные расходы — покрытие процентов
FCF / Долг — возможность погашения тела долга
Долг / Капитал — структура финансирования
```

### 5C кредитного анализа
| C | Английский | Смысл |
|---|---|---|
| **Character** | Репутация | Кредитная история, деловая репутация менеджмента |
| **Capacity** | Способность | Достаточность денежных потоков для обслуживания долга |
| **Capital** | Капитал | Размер собственных средств, «кожа в игре» |
| **Collateral** | Залог | Что можно забрать при дефолте |
| **Conditions** | Условия | Макро-, отраслевая среда, цель кредита |

### Кредитные рейтинги
| Агентство | Инвестиционный | Спекулятивный (мусор) |
|---|---|---|
| S&P / Fitch | AAA → BBB− | BB+ → D |
| Moody's | Aaa → Baa3 | Ba1 → C |
| АКРА / Эксперт РА | AAA → BBB− (ru) | BB+ (ru) → D (ru) |

## Блок 9. Анализ для банков (специфика)

Для банков стандартные метрики не работают — нет понятия «выручка» в классическом смысле, долг = продукт, а не риск.

```
NIM = NII / Средние доходные активы — ключевая маржа банка

CIR (Cost-to-Income) = Операционные расходы / Операционные доходы
  Норма: < 50% — эффективный банк

CoR (Cost of Risk) = Отчисления в резервы / Средний кредитный портфель
  Показывает стоимость кредитного риска

NPL ratio = Кредиты 3 стадии (NPL) / Кредитный портфель
Coverage ratio = Резервы / NPL  (должен быть > 100%)

ROE банка = Чистая прибыль / Средний капитал
ROA банка = Чистая прибыль / Средние активы  (норма: 1–2%)

Leverage банка = Активы / Капитал  (обычно 10–15x — это нормально)
```

### CAMELS — рейтинговая система для банков
| Буква | Критерий |
|-------|----------|
| **C** — Capital adequacy | Достаточность капитала (Н1) |
| **A** — Asset quality | Качество активов (NPL, Coverage) |
| **M** — Management | Качество управления |
| **E** — Earnings | Доходность (ROE, ROA, NIM, CIR) |
| **L** — Liquidity | Ликвидность (LCR, NSFR, Н2, Н3) |
| **S** — Sensitivity | Чувствительность к рыночным рискам (процентный риск, валютный) |

## Блок 10. Анализ качества прибыли

Не вся прибыль одинаково ценна. Нужно понять, насколько прибыль:
- **Повторяемая** (recurring) или разовая (one-off)
- **Денежная** или бумажная (accrual)
- **Операционная** или финансовая

### Признаки низкого качества прибыли
- Прибыль растёт, а OCF падает
- Дебиторка растёт быстрее выручки (клиенты не платят)
- Частые «неповторяющиеся» списания
- Изменение учётной политики улучшает показатели
- Capex внезапно снижается (экономят на развитии ради краткосрочной прибыли)

### Accrual Ratio
```
Accrual Ratio = (Чистая прибыль − OCF) / Средние чистые активы
```
Чем ближе к нулю — тем выше качество прибыли (прибыль подкреплена деньгами).

## Блок 11. Горизонтальный и вертикальный анализ

### Горизонтальный (Trend Analysis)
Сравниваем показатели во времени:
```
Изменение = (Показатель_текущий − Показатель_базовый) / Показатель_базовый × 100%
CAGR = (Значение_конец / Значение_начало)^(1/n) − 1
```

### Вертикальный (Common Size)
Каждая строка P&L — как % от выручки:
```
Себестоимость / Выручка = ?%
Валовая прибыль / Выручка = Gross Margin
Операционные расходы / Выручка = ?%
```
Позволяет сравнивать компании разного масштаба.

## Блок 12. Сценарный анализ и чувствительность

### Сценарный анализ
Строим 3 сценария — базовый, оптимистичный, пессимистичный — с разными допущениями:
- Темп роста выручки
- Маржинальность
- Стоимость долга / WACC
- Курс валюты / цена сырья

### Анализ чувствительности (Sensitivity / Tornado)
Меняем одну переменную, остальные фиксируем → смотрим, как меняется целевой показатель (DCF-оценка, NPV).

## Блок 13. Анализ инвестиционных проектов (Project Finance)

```
NPV (Net Present Value) = Σ CF_t / (1+r)^t − I_0
  NPV > 0 → проект создаёт стоимость

IRR (Internal Rate of Return) — ставка, при которой NPV = 0
  IRR > WACC → принимаем проект

Payback Period — срок окупаемости
  Discounted Payback — с учётом дисконтирования

PI (Profitability Index) = PV денежных потоков / Начальные инвестиции
  PI > 1 → проект эффективен
```

## Быстрые формулы-шпаргалка

```
Gross Margin      = (Выручка − COGS) / Выручка
EBITDA Margin     = EBITDA / Выручка
Net Margin        = ЧП / Выручка
ROE               = ЧП / Капитал
ROA               = ЧП / Активы
ROIC              = NOPAT / (Капитал + Чистый долг)
Current Ratio     = Оборотные активы / Краткосрочные обязательства
Quick Ratio       = (ОА − Запасы) / КО
Net Debt          = Долг − Денежные средства
Net Debt/EBITDA   = Чистый долг / EBITDA
ICR               = EBIT / % расходы
CCC               = DIO + DSO − DPO
FCF               = OCF − Capex
EV                = Капитализация + Чистый долг
EV/EBITDA         = EV / EBITDA
P/E               = Цена акции / EPS
WACC              = Ke × E/V + Kd × (1-t) × D/V
DCF               = Σ FCFF / (1+WACC)^t + TV/(1+WACC)^n
NPV               = Σ CF/(1+r)^t − I_0
```

## Распространённые ошибки в финансовом анализе

| Ошибка | Почему опасна |
|--------|--------------|
| Смешивать EBIT и EBITDA | Разные показатели — нельзя сравнивать мультипликаторы |
| Использовать P/E для банков | Некорректно — лучше P/B и ROE |
| Игнорировать разницу в учётной политике | Компании могут по-разному считать выручку / капитализировать расходы |
| Считать высокий долг всегда плохим | Для зрелых компаний с предсказуемым FCF долг — инструмент, а не риск |
| Путать номинальный и реальный рост | В DCF нужно или реальный FCFF + реальный WACC, или номинальный + номинальный |
| Забывать про minority interest в EV | EV должен включать долю меньшинства, т.к. консолидирует 100% активов |
